Кэширующий dns сервер
Версия 1.0 (09.05.2020)

Автор: Муравейник Алексей Олегович (alexmur20000@mail.ru). Группа КН-202 (МЕН-280207)


Описание
    Данное приложение является реализацией dns сервера с функцией кэширования. При запуске инициализируется кэш
    из ранее сохраненного файла backup, если такой файл отсутствует, то он будет позже создан. Далее программа начинает
    прослушивание порта 53. Сервер принимает и отправляет сообщения по протоколу DNS. При поступлении запроса от клиента,
    например через утилиту nslookup, происходит проверка на наличие запрашиваемой записи в кэше. В этот же момент происходит
    проверка на актуальность хранящейся информации (используется поле ttl, полученное при запросе записи от внешнего dns сервера).
    Если запись актуальна, то она передается клиенту с измененным значением ttl (уже с учетом того времени, которое
    запись лежала в кэше). Если запись устарела, то она удаляется из кэша, и выполняется новый запрос на внешний dns сервер.
    Если нужной записи нет, то выполняется новый запрос на внешний dns сервер. В данном примере используется авторитетный
    dns сервер, отвечающий за e1.ru. В его ответе присутсвует не только запрашиваемая информация, но также и информация о
    ns серверах и дополнительные записи (записи типа A для ns серверов). Все эти данные обрабатываются и кэшируются, затем
    отправляется ответ пользователю.
    Если по каким-то причинам внешний сервер не ответил в течении 2 секунд (например на этом dns сервере отсутствует интернет),
    то клиенту так же ничего не отправится, и сервер снова начнет ожидать входящее подключение

Состав:
    server.py - главный скрипт, запускающий сервер и выполняющий все действия по обработке
    utils.py - utility-class с полезными функциями
    answer.py - модель хранения ресурсных записей

Требования
    Python версии 3


Использование приложения

    Пример запуска:
            python server.py
